**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [AWS S3 service learning, project ideation and team planning](1.2-week2/)

**Week 3:** [DevOps fundamentals, CI/CD concepts, and EC2 networking](1.3-week3/)

**Week 4:** [Deep dive into AWS S3 service and lab practice](1.4-week4/)

**Week 5:** [AWS security model and organization services](1.5-week5/)

**Week 6:** [Project proposal completion, SES study, and Redis implementation](1.6-week6/)

**Week 7:** [CI/CD pipeline repair and VM installation scripts](1.7-week7/)

**Week 8:** [Mid-test preparation and reviewing all labs](1.8-week8/)

**Week 9:** [Well-Architecture Framework and API documentation](1.9-week9/)

**Week 10:** [Architecture analysis and AWS deployment with ClickOps](1.10-week10/)

**Week 11:** [Infrastructure as Code with Terraform and React frontend](1.11-week11/)

**Week 12:** [POC creation, unit testing, and performance optimization](1.12-week12/)
